# The Monomyth: What Campbell Actually Argued

This companion directory supports the article **"The Monomyth: What Campbell Actually Argued."**

The workflows operationalize monomyth analysis as a reviewable interpretive system. They include monomyth-pattern scoring, specificity-preservation diagnostics, formula-drift review, interpretation-readiness checks, source and authority review, counterexample tracking, ethical governance, markdown governance queues, and Catalyst Canvas-ready JSON exports.

## Core Claim

Campbell's monomyth is best understood as a symbolic model of heroic transformation through departure, initiation, and return. It is not a universal law of storytelling, not a screenwriting template, and not a substitute for source-specific myth scholarship. Responsible use requires source context, cultural specificity, ritual awareness, counterexamples, method limits, and ethical review.

## Repository Features

- Catalyst Canvas manifest and JSON schemas
- Monomyth claims, source contexts, pattern features, specificity notes, formula-drift risks, and governance datasets
- Python package-style audit workflow
- Base R diagnostic workflow
- SQL schema for monomyth claim analysis
- HTML/CSS/PHP/Java scaffolds for Canvas integration
- Markdown governance queues
- JSON Canvas cards
- Synthetic datasets and generated output folders
- Documentation for Campbell’s argument, departure-initiation-return, symbolic reading, formula drift, critiques, responsible use, and governance notes

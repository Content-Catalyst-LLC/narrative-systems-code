# Storytelling and the Architecture of Meaning Over Time

This companion directory supports the article **"Storytelling and the Architecture of Meaning Over Time."**

The workflows operationalize storytelling as temporal meaning architecture. They include temporal coherence audits, memory durability scoring, continuity and rupture mapping, narrative drift risk, revision priority scoring, archive and context-retention analysis, temporal governance queues, and Catalyst Canvas-ready JSON exports.

## Core Claim

Storytelling builds architecture for meaning over time. Stories connect origin, sequence, continuity, rupture, repetition, revision, memory, identity, and future projection. Meaning does not maintain itself; it must be preserved, questioned, revised, and governed.

## Repository Features

- Catalyst Canvas manifest and JSON schemas
- Meaning architecture datasets
- Memory layers, continuity and rupture maps, drift flags, revision priorities, and temporal governance notes
- Python package-style audit workflow
- Base R diagnostic workflow
- SQL schema for temporal narrative analysis
- HTML/CSS/PHP/Java scaffolds for Canvas integration
- Markdown governance queues
- JSON Canvas cards
- Synthetic datasets and generated output folders
- Documentation for temporal coherence, narrative drift, memory durability, revision governance, and responsible narrative maintenance

## Quick Start

From this article directory:

```bash
python3 python/run_meaning_architecture_canvas_audit.py
Rscript r/run_all_meaning_architecture_workflows.R
```

Outputs are written to:

```text
outputs/tables/
outputs/json/
outputs/markdown/
outputs/figures/
```

# Performance, Memory, and Variation in Oral Storytelling

This companion directory supports the article **"Performance, Memory, and Variation in Oral Storytelling."**

The workflows operationalize oral storytelling variation as a reviewable performance-memory system. They include performance-context scoring, memory-support diagnostics, variation-accountability review, formula and repetition tracking, audience and occasion documentation, transmission-pathway mapping, archive-risk scoring, platform-circulation review, governance queues, and Catalyst Canvas-ready JSON exports.

## Core Claim

Oral storytelling is not a defective version of writing. It is a living performance practice in which memory, formula, audience, occasion, voice, gesture, repetition, and variation help stories remain recognizable while adapting to new contexts.

## Repository Features

- Catalyst Canvas manifest and JSON schemas
- Oral storytelling variation, performance context, memory support, variation pattern, archive-risk, and governance datasets
- Python package-style audit workflow
- Base R diagnostic workflow
- SQL schema for oral storytelling performance and variation analysis
- HTML/CSS/PHP/Java scaffolds for Canvas integration
- Markdown governance queues
- JSON Canvas cards
- Synthetic datasets and generated output folders
- Documentation for performance context, memory support, variation tracking, transmission pathways, archive risk, and responsible use

## Quick Start

From this article directory:

```bash
python3 python/run_oral_storytelling_variation_canvas_audit.py
Rscript r/run_all_oral_storytelling_variation_workflows.R
```

```text
outputs/tables/
outputs/json/
outputs/markdown/
outputs/figures/
```
